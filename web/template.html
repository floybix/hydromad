introduction

hydromad is an R package (i.e. a software package for the R statistical computing environment), providing:

  • Simulation, estimation, assessment and visualisation of catchment response to time series of rainfall and other drivers.
  • Spatially-lumped models and tools for environmental hydrology (focusing on rainfall and streamflow time series data).
  • A modelling framework for water balance accounting and flow routing.

The IHACRES framework is used, where areal rainfall is passed through a soil moisture accounting model to estimate effective rainfall; this is then passed through a routing model to estimate streamflow. Included are mature implementations of models designed for this framework, notably the IHACRES CWI and CMD soil moisture accounting models, and unit hydrograph transfer functions for the routing.

This site serves as a reference for the R functions and datasets in the package. A gentler introduction can be found in the vignettes hydromad Tutorial and Data checking with hydromad. The package is described in the paper

hydromad: an open software environment for top-down modelling of catchment hydrology.

The package is maintained by Felix Andrews. The general modelling approach is based on work by Tony Jakeman and Barry Croke at The Australian National University.
